# Chicago 'L' Intelligent Transit Assistant

A machine learning system that predicts CTA L train delays and explains them conversationally using a custom self-attention model and Claude AI.

**Live demo:** https://cta-assistant.streamlit.app

---

## What It Does

Select a CTA L station, time of day, weather condition, and whether there's a major sporting event nearby. The attention model predicts how many minutes of delay to expect and shows exactly which factor is driving it — with attention weights as percentages. Claude then explains the situation in plain English and gives you an actionable recommendation.

---

## System Architecture

**Stage 1 — Attention Model**

A custom multihead self-attention model built in PyTorch. Each of four input features is treated as an independent token rather than concatenated into a flat vector:

- `station_id` — which station on the L system
- `time_norm` — time of day normalized to 0–1
- `weather_index` — composite score from 0 (clear) to 1 (blizzard)
- `sports_flag` — binary indicator for game-day proximity to a venue

The transformer block uses 4 attention heads over a 32-dimensional sequence and outputs both a delay prediction in minutes and the attention weights per feature. The column mean of the attention matrix gives a per-feature importance score showing what drove the prediction — during a blizzard Weather Index hits 57%, during a Sox game Sports Event hits 54%.

**Stage 2 — Prompt Builder**

The attention model outputs are formatted into a structured prompt: line, station, time, weather label, delay estimate, and top attention features by percentage.

**Stage 3 — Claude API**

The structured prompt is passed to Claude (claude-3-5-sonnet) which returns a 3–4 sentence advisory explaining the situation and giving a concrete recommendation — including backup route suggestions when relevant.

---

## Data Sources

**Sports venues** — Hardcoded GPS coordinates for Wrigley Field (Cubs), Guaranteed Rate Field (White Sox), Soldier Field (Bears), and United Center (Bulls/Blackhawks). Stations within proximity are tagged with a sports flag. Game start times are fetched via API and the flag is active for the 3-hour pre/post game window. A ridership multiplier derived from game-day vs. non-game-day historical data quantifies crowd impact.

**Weather index** — Composite 0–1 score from snow depth, precipitation, wind speed, and temperature, each weighted by its known impact on CTA L operations.

**Weather bias prior** — Weather effects cannot be reliably learned from ridership data alone. A piecewise linear lookup table built from published research (TCRP Report 165) maps weather conditions to added delay minutes — 0 min for clear conditions up to 22 min for an extreme blizzard.

**Ridership** — 2022–23 daily per-station CTA ridership distributed across time-of-day profiles. Delay modeled as a function of crowd density using the TCRP Report 165 dwell-time formula.

**CTA Train Tracker API** — Live arrival predictions for all 108 CTA L stations, used to render the live system map in the app.

## Model Details

| Property | Value |
|----------|-------|
| Architecture | Feature-as-token multihead self-attention |
| Input features | 4 (station_id, time_norm, weather_index, sports_flag) |
| Attention heads | 4 |
| Embedding dimension | 32 |
| Output | delay_minutes + attention weights |
| Loss function | Huber loss |
| Validation MAE | ~0.47 min |
| Validation RMSE | ~0.66 min |
| Convergence | Epoch 2 |

Trained weights are saved at `models/transit_attention.pt`.

The key design decision was treating features as tokens rather than concatenating them. This lets self-attention discover cross-feature interactions automatically — for example, the combination of high `sports_flag` and rush hour is superadditive, producing more delay than either factor would predict alone.

## LLM Fine-Tuning

| Property | Value |
|----------|-------|
| Base model | meta-llama/Meta-Llama-3.1-8B-Instruct |
| Method | QLoRA (0.35% of 8B parameters trained) |
| Training examples | 1,291 train / 144 eval |
| Format | Llama 3.1 chat JSONL (system/user/assistant) |
| Training | Unsloth on Google Colab T4 GPU, 3 epochs |
| Final loss | 0.150 → 0.137 |
| Adapter | https://huggingface.co/jiya304/cta-llama-adapter |

Training data was generated by running synthetic scenarios through the attention model, collecting predictions and attention weights, and formatting them as chat examples. Training data is at `data/llama_train.jsonl`.

> **Note on deployment:** The Llama adapter requires a GPU. Streamlit Community Cloud has no GPU so the deployed app uses the Claude API with the same structured prompt format. The adapter is available for local GPU inference — see Running Locally below.

## App Features

- Station selector — all 108 CTA L stops
- Time of day slider — hour and minute
- Weather dropdown — clear through blizzard
- Sports event toggle — game-day proximity flag
- Live CTA system map — all stations colored by line
- Delay status badge — on time or delayed with minutes

## Running Locally

**Requirements:** Python 3.10+

```bash
git clone https://github.com/your-repo/cta-ai-assistant
cd cta-ai-assistant
pip install -r requirements.txt
```

Copy `.env.example` to `.env` and fill in your API keys:

```bash
cp .env.example .env
```

```
CTA_API_KEY=your_cta_api_key_here
OPENWEATHER_API_KEY=your_openweather_api_key_here
HF_TOKEN=your_huggingface_token_here
ANTHROPIC_API_KEY=your_anthropic_api_key_here
```

Then run:

```bash
streamlit run app.py
```

**To use local Llama inference instead of Claude API**, you need a GPU and unsloth installed:

```bash
pip install unsloth
```

Then set `LLAMA_ADAPTER_PATH=models/llama_cta_adapter` in your `.env` and swap the inference function in `app.py` to load from the local adapter or from `jiya304/cta-llama-adapter` on HuggingFace.
